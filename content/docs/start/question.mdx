# 常见问题

<Accordions>
<Accordion title="Ech0 是什么？">
Ech0 是一款轻量级的开源自托管平台，专为快速发布与分享个人想法、文字和链接而设计。它提供简洁的界面，支持零干扰的写作体验，所有数据存储于本地，确保用户对内容的完全控制。
</Accordion>

<Accordion title="Ech0 不是什么？">
Ech0 不是传统的笔记软件，设计之初并不是为了专业的笔记管理和记录（如 Obsidian、Notion 等），Ech0 的核心功能类似朋友圈/说说。
</Accordion>

<Accordion title="Ech0 是免费的吗？">
是的，Ech0 完全免费且开源，遵循 AGPL-3.0 协议。它没有广告、追踪、订阅或服务依赖。
</Accordion>

<Accordion title="如何进行备份和导入数据？">
由于所有内容都存储在本地 SQLite 文件中，建议定期备份 `/opt/ech0/data` 目录（或您部署时映射的数据目录）。在线数据管理中支持"快照导出"和"迁移导入"：导出用于离线归档，导入统一通过迁移功能完成。
</Accordion>

<Accordion title="Ech0 支持 RSS 吗？">
是的，Ech0 支持 RSS 订阅，您可以通过 RSS 阅读器订阅您的内容更新。
</Accordion>

<Accordion title="为什么发布失败，提示联系管理员？">
当前版本设计上，只有管理员可以发布内容。部署后，首个注册的用户会自动被设置为系统管理员，其他用户无法发布内容（可在设置中分配权限）。
</Accordion>

<Accordion title="Ech0 的权限模型是怎样的？">
Ech0 提供四级权限角色：**Owner**（最高权限，管理系统设置、管理员和所有用户）、**Admin**（管理内容、评论、文件和部分设置）、**User**（管理自己的 Echo 和文件）、**Guest**（浏览公开内容、点赞和评论）。首个注册用户自动成为 Owner，Owner 可在用户管理中为其他用户分配管理员权限。系统默认最多 5 个用户账号。
</Accordion>

<Accordion title="为什么别人无法显示自己的 Connect 头像？">
要使别人显示自己的 Connect 头像需要在`系统设置-服务地址`中填入自己当前的实例地址，比如我自己填的是部署 ech0 后的域名`https://memo.vaaat.com`（注意：这里填的链接需要带上 http 或 https）。
</Accordion>

<Accordion title="设置中的 MetingAPI 项是什么？">
这是用于解析获取音乐流媒体直链的服务 API，用于分享的音乐卡片功能，如果不设置则默认使用 Ech0 提供的 API（部署于 Vercel）。
</Accordion>

<Accordion title="为什么添加后的 Connect 只显示了一部分？">
因为后端会尝试获取所有 Connect 的实例信息，如果某个实例挂了或者无法访问则会被抛弃，只返回获取到的有效 Connect 实例的信息给前端。
</Accordion>

<Accordion title="Ech0 不建议发什么？">
Ech0 发布的内容分为三部分：文字、图片、扩展内容（如音乐、视频等播放器卡片），Ech0 不建议发布同时包含`文字 + 图片 + 扩展内容`这种密集内容，因为其违反了 Ech0 的一些设计理念，同时在任何时候都不推荐发布扩展内容或长篇幅的文章。
</Accordion>

<Accordion title="如何开启评论功能？">
在 **系统设置 → 评论设置** 中启用内建评论功能即可。可选配置是否需要审核、是否启用 CAP 验证码。Ech0 使用完全内建的原生评论系统，无需任何第三方评论服务。
</Accordion>

<Accordion title="S3 存储如何配置？">
在存储设置页面填入所需配置信息，注意：endpoint 不需要填 http 或者 https 开头，存储桶需提供公共访问权限。
</Accordion>

<Accordion title="如何启用 Passkey 无密码登录？">
在 **SSO → Passkey** 页面先配置 WebAuthn RP ID（域名）与 WebAuthn Origins（站点完整 URL），保存并显示"Passkey 就绪"后，再按浏览器提示绑定你常用的生物识别或安全密钥设备即可使用。详见 [Passkey 指南](/docs/guide/passkey)。
</Accordion>

<Accordion title="如何使用 AI Agent 摘要功能？">
在 **系统设置 → Agent** 中启用 Agent，选择 LLM 提供商（OpenAI/DeepSeek/Claude/Gemini/Qwen/Ollama/Custom），填写 API Key 和模型名称即可。摘要结果通过 `/api/agent/recent` 公开访问。
</Accordion>

<Accordion title="Zone 热敏打印台是什么？">
Zone 是 Ech0 独有的创意功能，模拟热敏打印机将 Echo 内容"打印"为可拖拽的纸质卡片。访问 `/zone` 体验，卡片位置自动保存到浏览器本地存储。
</Accordion>

<Accordion title="如何调用 Ech0 API？">
通过 **系统设置 → 访问令牌** 生成 Token，在请求头中携带 `Authorization: Bearer <token>`。完整接口文档见 `/swagger/index.html`。
</Accordion>
</Accordions>
